# 入力フォーマット仕様

## CSV形式

`period,item,value` の3列ロング形式。1行が「ある期のある勘定科目の値」を表す。

```csv
period,item,value
FY2022,売上高,500000000
FY2022,売上原価,350000000
FY2023,売上高,480000000
FY2023,売上原価,340000000
```

- `period`: 期を表す文字列で、**4桁の年を一意に1つだけ含むこと**（例: `FY2022`, `2023年3月期`）。CSV内の記載順は問わない（後述）
- `item`: 勘定科目名。表記ゆれは下記の対応表で正規化される
- `value`: 数値。単位は円でも千円でも構わないが、**同一ファイル内で単位を統一**すること。半角/全角のカンマ区切りや全角数字（例: `500,000`, `５００，０００`）も自動的に正規化されるので使ってよい
- 文字コードはUTF-8（BOM付きも許容）

テンプレート: `assets/input-template.csv`

## 期間の時系列順について

**時系列順（古い期→新しい期）は `period` ラベルに含まれる4桁の年から自動判定する。CSVの記載順は時系列の根拠として使わない。**

決算書エクスポートは最新期が先頭に来ることも多く、CSVの行順を時系列の根拠にすると、記載順が新しい期からだった場合に前期比・成長率が黙って逆向きに計算されてしまう（実際に踏んだ回帰バグ）。これを避けるため:

- `period` ラベルに4桁の年（`FY2022` の `2022`、`2023年3月期` の `2023` 等）が一意に1つ含まれていれば、その年の昇順で自動的に並べ替えて計算する
- CSVの記載順と時系列順が異なっていた場合、`ratios.json` に `periods_order_note` フィールドが追加され、並べ替えた旨と元の記載順が記録される（黙って並べ替えない）
- `令和5年度` のように4桁の年を一意に抽出できないラベルや、複数の期間ラベルが同じ年を指す場合は、推測せず `ValidationError` で止まる。その場合は `FY2023` のように4桁の年を含む一意なラベルに書き換えること

## 期間数

- 最低1期から動作するが、前期比・成長率・ベンチマーク差分の一部指標は前期データがないと算出不可（`value: null`, `note: "前期データがないため算出不可"` として出力される）
- 2〜3期分を推奨（診断士実務でも直近2〜3期を見るのが標準）

## 勘定科目マッピング対応表

`validate_input.py` の `ACCOUNT_ALIASES` がこの対応表の実体。以下のいずれかの表記であれば自動的に正規科目名に正規化される。

| 正規科目名 | 許容される別名 |
|---|---|
| 売上高 | 売上高, 売上収益, 営業収益, 売上高合計 |
| 売上原価 | 売上原価 |
| 販売費及び一般管理費 | 販売費及び一般管理費, 販管費 |
| 営業利益 | 営業利益 |
| 営業外収益 | 営業外収益 |
| 営業外費用 | 営業外費用 |
| 経常利益 | 経常利益 |
| 当期純利益 | 当期純利益, 当期純利益金額, 親会社株主に帰属する当期純利益 |
| 現金及び預金 | 現金及び預金, 現金預金 |
| 売上債権 | 売上債権, 受取手形及び売掛金, 売掛金 |
| 棚卸資産 | 棚卸資産, 商品及び製品, 商品 |
| 流動資産合計 | 流動資産合計, 流動資産 |
| 有形固定資産 | 有形固定資産 |
| 固定資産合計 | 固定資産合計, 固定資産 |
| 資産合計 | 資産合計, 総資産 |
| 流動負債合計 | 流動負債合計, 流動負債 |
| 固定負債合計 | 固定負債合計, 固定負債 |
| 負債合計 | 負債合計, 負債 |
| 純資産合計 | 純資産合計, 純資産, 自己資本 |

**上記のどれにも一致しない科目名がある場合、`validate_input.py` は推測せずエラーとして列挙する。** その場合はCSV側の科目名を正規科目名（または対応表の別名）に書き換えて再実行する。新しい別名を対応表に追加したい場合は `scripts/validate_input.py` の `ACCOUNT_ALIASES` を編集する。

**同一期間・同一正規科目に複数の入力行がマッピングされた場合もエラーになる。** 例えば「受取手形及び売掛金」と「売掛金」はどちらも「売上債権」に正規化されるため、同じ期に両方の行があると、どちらが正しいか推測できず `ValidationError` になる。重複行を削除・統合してから再実行すること。

## 必須科目

全19科目（上表の正規科目名すべて）が各期に揃っている必要がある。不足があればエラーで列挙される。棚卸資産や有形固定資産がない業態（無在庫のサービス業等）でも、0として明示的に行を用意すること。

## 損益項目の整合性チェック

`販売費及び一般管理費`・`営業外収益`・`営業外費用` はどの指標計算にも直接使われないが、入力データの検算に使われる。以下の関係が成立しているかを確認し、成立していなければ入力ミスの可能性があるとして `ValidationError` で止まる（丸め誤差程度の差は許容する）:

- `営業利益 ≈ 売上高 - 売上原価 - 販売費及び一般管理費`
- `経常利益 ≈ 営業利益 + 営業外収益 - 営業外費用`

## PDF入力について

Phase 1ではスコープ外。決算書PDFからCSVへの変換は手動またはユーザー側のツールに委ねる。
